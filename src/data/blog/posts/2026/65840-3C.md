## 1 实验目标

实现 Raft 的持久化功能，确保服务器在崩溃重启后能够恢复状态并继续工作。

## 2 核心实现细节

### 2.1 需要持久化的状态字段

> The paper's Figure 2 mentions which state should be persistent.

![Figure 2](@/assets/images/posts/2026/65840-3C/raft_state.png "Figure 2")

图中显示需要持久化的状态有：

- `currentTerm` - 当前任期。服务器所见过的最新任期（首次启动时初始化为 0，随后单调递增）。
- `votedFor` - 当前任期投票给的候选人。当前任期中获得投票的候选者 ID（若无则为 null）。
- `log[]` - 日志条目。每个条目包含状态机的命令以及该条目被领导者接收时的任期（起始索引为 1）。

```go
type Raft struct {
    // ... 其他字段

    // 持久化状态 (所有服务器)
    currentTerm int        // 最新任期
    votedFor    int        // 当前任期投票给的候选人 (-1 表示未投票)
    log         []LogEntry // 日志条目

    // 易失性状态 (不需要持久化)
    // commitIndex, lastApplied, nextIndex, matchIndex...
}
```
